# GitHub profile README design

**Status:** approved for implementation

## Goal

Create a concise, trustworthy GitHub profile README for `vins-anity` with the
information density and terminal feel of neofetch.

## Structure

- Left: a static ASCII-art PNG made from the supplied portrait.
- Right: a terminal-style identity and stack summary.
- Below: three selected projects linked only where a public, verified destination exists.

## Public copy

- **Role:** Backend / Product Engineer
- **Focus:** TypeScript APIs, PostgreSQL and BigQuery workflows, AI automation
- **Builds:** product systems for lead qualification, operations, and decisions
- **Stack:** TypeScript, Bun, Node.js, React, SQL, BigQuery
- **Tooling:** OpenAPI, Vitest, Docker, GitHub Actions
- **Projects:** Simulark, EcoRoute, Chronosphere

## Constraints

- Do not claim unverified titles, employer names, metrics, or technologies.
- Do not include contact details or the full resume.
- Use no dynamic badges, contribution widgets, animations, or third-party trackers.
- Keep the implementation to `README.md` and `assets/profile-ascii.png`.
